# 物流配送路线优化方案

## Context

Daniel物流公司在SFO机场有UPS物流中心，需要给旧金山多个收货点进行货物配送。有10辆物流车（3种车型），需要计划最佳配送方案。

Mapbox 原生 API 仅支持单路线TSP优化，不支持多车辆和容量约束。因此需要结合 Mapbox API + VRP求解器。

## Features

### 核心功能
- **手动设置收货点**：点击地图添加收货点，完成设置后自动赋予随机收货量(1-10kg)
- **智能路线规划**：基于Mapbox Matrix API距离数据，使用贪心算法+2-opt优化
- **状态持久化**：使用 localStorage 保存收货点和方案，刷新/切换标签页后自动恢复
- **可视化高亮**：鼠标浮动到配送方案时高亮路线和对应收货点
- **颜色同步**：收货点Marker颜色与所属路线颜色一致

### 交互效果
- 鼠标浮动到配送方案时：
  - 路线线条加粗、高亮
  - 该路线配送的收货点Marker放大1.3倍
- 每辆车用不同颜色线条表示路线
- 收货点列表可折叠展示

## 算法说明

### VRP 求解算法
1. **贪心初始化**: 按需求降序排列收货点，分配给距离最近且容量充足的车辆
2. **容量约束**: 每辆车累计货物量不能超过容量（可允许50%超载）
3. **2-opt 优化**: 对每辆车的路线进行局部优化，减少总行驶距离
4. **Directions API**: 获取最终路线的详细几何形状用于地图绘制

### Mapbox API 使用
- **Matrix API**: 获取仓库到各收货点、各收货点之间的距离矩阵
- **Directions API**: 获取最终路线的详细几何形状用于地图绘制

## 安全说明

### 关于 Mapbox Access Token

本项目使用 **public token**（`pk.eyJ1...`）来初始化 Mapbox GL JS。

**Public Tokens：**
- 适用于客户端使用（浏览器 Web 应用）
- 仅应拥有 **public scopes**（如 styles:read 只读权限）
- **在客户端代码中可见** - 这是预期且设计如此
- 可通过 **URL 限制** 限制只能在指定域名下工作
- 适用于 Mapbox GL JS v0.53.1+

**Secret Tokens：**
- **绝对不能暴露给客户端**
- 需要 secret scopes 的请求应在服务器端进行
- 敏感操作可使用 **Tokens API** 创建临时 token

**最佳实践：**

1. **生成新 token** - 不要使用默认 public token，创建具有适当 scopes 的新 token
2. **设置 URL 限制** - 限制为你的特定域名
3. **最小化 scopes** - 只授予任务必需的 scopes
4. **隔离 tokens** - 每个应用使用不同 token 便于追踪
5. **轮换 tokens** - 如有可疑使用，立即轮换
6. **监控使用量** - 检查统计页面发现异常模式

详见 [Mapbox 安全文档](https://docs.mapbox.com/help/dive-deeper/how-to-use-mapbox-securely/#access-tokens)。
